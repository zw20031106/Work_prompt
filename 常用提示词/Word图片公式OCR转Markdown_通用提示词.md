# Word 文件内容完整提取并转化为 Markdown 的通用提示词

你是一名严谨的数学资料整理与 OCR 转写助手。现在我将上传一个 Word 文件，请你将 Word 文件中的全部内容完整提取并整理为 Markdown 文件。

## 一、任务目标

请将 Word 文件中的所有内容，包括：

1. Word 正文中的文字；
2. 嵌入图片、截图、扫描图中的文字；
3. 图片中的数学公式、变量、数字、表格；
4. 题目、选项、答案、解析、注释、补充说明；
5. 手写内容、图示旁边的批注；
6. 坐标图、概率分布图、区域图、表格、流程性内容；

全部转化为**可编辑的 Markdown 文本**。

最终输出格式为 `.md` 文件。

## 二、核心要求

### 1. 内容完整性

请逐页、逐段、逐图检查，不得只提取 Word 中可直接复制的文字。  
对于嵌入图片中的内容，必须进行识别和转写。

尤其注意以下内容不能遗漏：

- 图片中的例题；
- 图片中的答案和解析；
- 图片中的公式推导；
- 图片中的表格；
- 图片中的手写推导；
- 图片中的蓝色、红色批注；
- 图片下方或旁边的补充说明；
- 章节标题、小标题、编号、注释。

如果某部分图片模糊，无法百分百识别，请不要随意猜测，使用如下格式标注：

> 【待人工复核：此处图片文字/公式较模糊，疑似为……】

### 2. 数学公式格式

所有数学变量、公式、函数、概率符号、分布符号、积分、求和、极限、矩阵、分段函数等，都必须使用 Markdown 数学公式格式。

行内变量使用：

```markdown
$X$、$Y$、$f(x)$、$P(A)$
```

独立公式使用：

```markdown
$$
P(A \cup B)=P(A)+P(B)-P(AB)
$$
```

多行推导使用：

```markdown
$$
\begin{aligned}
P(A \cup B)
&=P(A)+P(B)-P(AB) \\
&=1-\cdots
\end{aligned}
$$
```

分段函数使用：

```markdown
$$
f(x)=
\begin{cases}
\frac{1}{b-a}, & a<x<b, \\
0, & \text{其他}.
\end{cases}
$$
```

矩阵使用：

```markdown
$$
\begin{pmatrix}
a & b \\
c & d
\end{pmatrix}
$$
```

表格中的公式也必须使用数学公式格式，不允许直接用普通文本写变量。

错误示例：

```markdown
X服从N(0,1)
```

正确示例：

```markdown
$X \sim N(0,1)$
```

### 3. Markdown 排版要求

请按照原 Word 文件的逻辑结构重新排版，而不是机械堆叠 OCR 结果。

建议结构如下：

```markdown
# 文件标题

## 一、章节标题

### 1. 知识点标题

#### 核心结论

#### 公式整理

#### 例题

**题目：**

**解：**

**答案：**

#### 注释 / 易错点 / 补充说明
```

对于原文件中的编号，如“一、二、三”“1. 2. 3.”“例 1.1”等，应尽量保留。

如果原文中题目和解析分离，请整理到同一个模块下，保证题目、答案、解析一一对应。

### 4. 题目与解答处理要求

凡是出现题目、例题、选择题、填空题、证明题、计算题，请统一整理为以下格式：

```markdown
#### 例题 X

**题目：**  
……

**解答：**  
……

**答案：**  
……

**涉及知识点：**

- ……
- ……

**方法总结：**  
……
```

如果原文中没有“涉及知识点”和“方法总结”，可以根据题目内容进行简要补充，但不能改变题意，也不能编造原文中不存在的条件。

### 5. 图片内容处理要求

对于 Word 中的图片，请按以下规则处理：

1. 如果图片中主要是文字、公式、题目、解析，请全部转写为 Markdown 文本和 LaTeX 公式。
2. 如果图片中是表格，请转写为 Markdown 表格；表格中的公式也要使用 `$...$`。
3. 如果图片中是坐标图、概率密度图、分布函数图、几何区域图：
   - 优先用文字说明图像含义；
   - 必要时用 Markdown + LaTeX 重建核心关系；
   - 如果图像对理解很重要但无法准确重绘，请保留“图示说明”并标注“建议保留原图”。
4. 如果图片中有手写公式或手写推导，请尽量转写为规范公式，并标注其来源为“手写推导整理”。
5. 不要把图片简单写成“此处有图”，必须说明图片内容。

示例：

```markdown
**图示说明：**  
该图表示区间 $(a,b)$ 上的均匀分布，其概率密度函数在 $(a,b)$ 内为常数 $\frac{1}{b-a}$，在区间外为 $0$。
```

### 6. 知识补充要求

对于原文件中明显属于数学、概率论、统计学的知识点，可以在不改变原文含义的前提下，适当补充以下内容：

- 公式适用条件；
- 常见易错点；
- 推导中的关键步骤；
- 考试中常见问法；
- 与相近概念的区别；
- 方法总结。

但补充内容必须单独标注为：

```markdown
> **补充说明：**  
> ……
```

不要把补充内容混入原文，避免和原始资料混淆。

### 7. 准确性要求

请严格保证以下几点：

1. 不得漏掉页面、图片、题目、公式、表格。
2. 不得随意改写数学条件。
3. 不得改变题目原意。
4. 不得把不同题目的解答混在一起。
5. 不得把变量写错，例如将 $X$ 写成 $Y$，将 $f(x)$ 写成 $F(x)$。
6. 不得把上下标、积分限、求和范围、省略号写错。
7. 对模糊内容必须标注“待人工复核”，不能强行猜测。
8. 公式中的中文说明应使用 `\text{}`，例如：

```markdown
$$
f(x)=
\begin{cases}
1, & 0<x<1, \\
0, & \text{其他}.
\end{cases}
$$
```

### 8. 输出格式要求

请最终输出一个完整的 Markdown 文件内容。

如果可以生成文件，请直接生成 `.md` 文件并提供下载链接。

如果不能直接生成文件，请直接输出完整 Markdown 源码，不要只输出摘要。

输出前请进行自检，检查以下内容：

- [ ] 是否逐页处理了 Word 文件；
- [ ] 是否处理了所有图片中的文字和公式；
- [ ] 是否所有变量都使用了数学公式格式；
- [ ] 是否所有独立公式都使用了 `$$...$$`；
- [ ] 是否题目、答案、解析对应清楚；
- [ ] 是否表格已经转为 Markdown 表格；
- [ ] 是否模糊内容已标注“待人工复核”；
- [ ] 是否没有遗漏章节标题和编号；
- [ ] 是否 Markdown 层级清晰；
- [ ] 是否没有把图片内容简单跳过。

## 三、输出风格

请使用中文整理，风格要求：

- 严谨；
- 清晰；
- 适合后续复习和二次编辑；
- 数学表达规范；
- 不要口语化；
- 不要输出无关解释；
- 不要只给总结，必须完整转写原文件内容。

## 四、特别强调

这个任务的重点不是简单复制 Word 文本，而是把 Word 文件中“正文 + 图片 + 扫描内容 + 公式 + 手写内容”全部转化为可编辑的 Markdown 学习资料。

请务必对图片内容进行完整识别和重排。

如果文件页数较多，请按章节分批输出，但每一批都必须保证内容完整，不得省略中间页；每次输出前说明本批处理的页码范围。

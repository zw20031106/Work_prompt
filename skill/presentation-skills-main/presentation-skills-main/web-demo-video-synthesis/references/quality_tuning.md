# 清晰度与参数调优（timeline 录屏 + 后期字幕烧录）

## 四个容易混淆的量（决定“真清晰度”）

1. `viewport`
- 浏览器页面布局尺寸。
- 影响页面在录制时“看到多少内容”。

2. `record-size`
- Playwright 输出视频帧尺寸。
- 决定原始视频像素分辨率上限。

3. `deviceScaleFactor`
- 浏览器渲染像素密度。
- 提高可提升文字和细线清晰度，但会增加资源占用。

4. 输出转码分辨率（`width` / `height`）
- 最终 MP4 的画布尺寸。
- 如果高于原始有效像素，不会“凭空变清晰”。

## 常见误区（这几个坑我们踩过）

- 把 `viewport` 拉到 4K：通常只是“画布更大、网页更小”，细节未必更清晰。
- 只提高码率/换封装（WebM -> MP4）：只能减少压缩损失，不会修复低细节源。
- 录制参数看起来是 4K：不代表“内容细节是 4K”。关键仍是渲染密度（dsf）与录制有效像素（record-size）。
- 以为“视频 DPI”：视频没有 DPI 这个概念。要看像素、缩放策略、播放窗口 1:1 对比。

## 推荐组合

### 常规高清（1080p）
- viewport: `1920x1080`
- record-size: `1920x1080`
- deviceScaleFactor: `1` 或 `1.25`
- transcode: `1920x1080`, `crf=18~22`

### 高质量演示（2K）
- viewport: `1920x1080`
- record-size: `2560x1440`
- deviceScaleFactor: `1.5~2`
- transcode: `2560x1440`, `crf=16~20`

### 超高清演示（4K）
- viewport: `1920x1080`（保持常见演示视野与布局）
- record-size: `3840x2160`
- deviceScaleFactor: `2`
- transcode: `3840x2160`, `crf=16~20`

### 真 4K（推荐默认）

目标：在不改变“画面范围/元素相对大小”的前提下提高清晰度。

- `viewport=1920x1080`
- `deviceScaleFactor=2`
- `record-size=3840x2160`
- 转码保持 `3840x2160`

这个组合通常比“直接把 viewport 拉到 4K”更稳，因为后者容易出现“网页变小、信息密度过高”的观感问题。

## 关键结论

- 单纯“转 MP4”不会提升清晰度。
- 单纯“加码率”也不会修复低分辨率源。
- 清晰度优先级通常是：
  1) 录制有效像素（record-size + dsf）
  2) 页面布局是否占满可视区（viewport）
  3) 转码参数（CRF、maxrate）

## 最小验收清单（建议每次都做）

1. 用工具确认输出尺寸与预期一致（例如 3840x2160）。
2. 抽取 2-3 个关键帧，人工检查文字边缘是否有明显锯齿/糊边。
3. 核对“页面占画面比例”：避免出现“小网页贴在大画布”的情况。
4. 确认字幕框未越界，且不遮挡关键交互区域。

## 字幕安全区建议

- 字幕放底部居中（`Alignment=2`）。
- 保留底边距（`MarginV` 视 UI 底部栏高度而定；通常 `18~54` 之间微调）。
- 字体大小不要从 UI 感觉“差不多”开始猜：先出 1 版成片，再按实际遮挡/越界调整。
- 如果网页需要放大（zoom），字幕样式要预留安全区，避免左右越界或底部被遮挡。

## 字幕换行与显示区（重点经验）

### 先区分两种换行

1. **硬换行**（SRT 内已有 `\n`）
- 来源通常是 `build_srt_from_timeline.py` 的 `--wrap-max-units` 太小，或设置了 `--wrap-max-lines 2`。
- 一旦硬换行写进 SRT，后续只改 `MarginL/R` 常常“看起来没变化”，因为每行长度已经被锁死。

2. **软换行**（SRT 不写 `\n`，交给渲染器按可用宽度换行）
- 更适合横屏讲解视频，能更充分利用可显示宽度。

### 横屏推荐调参顺序（避免反复返工）

1. 先固定 `FontSize` 到“你觉得好听且好读”的大小。
2. 再减小 `MarginL/MarginR` 扩大可显示区。
3. 再增大 `--subtitle-wrap-max-units`（减少过早预换行）。
4. 仍然频繁换行时，再拆分文案或微降字号。

### 可复用起步参数（按画幅，不按语言）

- 竖屏（9:16，1080x1920）：
  - `FontSize=10~12`
  - `MarginV=28~72`
  - `MarginL/R=96~140`
  - `--subtitle-wrap-max-units 32~40`
  - 说明：竖屏必须优先保证左右安全区，宁可多一点换行。

- 横屏（16:9，4K 输出）：
  - `FontSize=13~16`
  - `MarginV=18~36`
  - `MarginL/R=8~40`
  - `--subtitle-wrap-max-units 140~180`
  - `--subtitle-wrap-max-lines 0`
  - 说明：横屏优先减少“碎行”，不建议默认强制两行上限。

## 竖屏（9:16）专项建议

- 录制参数优先从 `viewport=1080x1920`、`record-size=1080x1920` 起步。
- 竖屏字幕不要沿用横屏边距：必须显式增大 `MarginL/MarginR`，否则很容易左右越界。
- 常见起步值：`FontSize=10~12`、`MarginV=28~72`、`MarginL=96~140`、`MarginR=96~140`。
- 结合自适应换行：`--subtitle-auto-wrap true --subtitle-wrap-max-units 32~40`。
- 字幕越界时，优先增大左右边距，其次减小字号，再调小 `wrap-max-units`，最后再拆分过长文案。

## 翻页节奏与页面信息量

- 想要“讲完一页再翻页”，网页结构应是“一页一主题”，而不是把全部内容堆在一个长页面顶部。
- 每个主题 section 建议至少 `min-height: 100vh`，保证滚动到锚点时有明显换页感。
- 每页保持一个视觉重心（大标题 + 关键要点），减少密集段落导致的阅读压力。

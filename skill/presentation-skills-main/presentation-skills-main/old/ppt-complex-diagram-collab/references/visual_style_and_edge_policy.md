# 视觉方案与连线策略（经验总结）

本文把我们在 demo 迭代中遇到的“常见翻车点”与可复用的解决思路固化下来，作为 `ppt-complex-diagram-collab` 的默认视觉与结构策略参考。

目标：**让图在第一眼可读**、在第二眼可解释、在实现上可复现；并且尽量避免“看起来复杂但信息密度为零”的假复杂。

---

## 1. 我们遇到的典型问题

### 1.1 连线过多，变成 hairball
- 症状：一页里边数接近节点数甚至超过节点数很多；读者无法在 3 秒内识别主路径。
- 根因：把“依赖/关联/可能影响/可调用”都画成同一类边；没有区分主线与支线；缺少边预算（edge budget）。

### 1.2 连线语义不清，出现“无意义的线”
- 症状：线存在但回答不了任何读者问题；线只是在“证明我很复杂”。
- 根因：边没有承载明确语义（例如：数据流、控制流、治理审批流、反馈闭环），也没有文本/图例支撑。

### 1.3 颜色“拍脑袋”，导致可读性差
- 症状：整体偏浅/偏花；文字在某些底色上发虚；同一层级颜色不一致。
- 根因：用亮度阈值或主观感觉选文字色；没有对比度指标；主色饱和度与面积控制不当。

### 1.4 箭头/层级表达不成体系
- 症状：chevron 文案过长、方向不一致；箭头抢戏或像装饰；层级递进关系不稳定。
- 根因：没有选定“视觉语法（visual grammar）”：到底是因果图/数据流图，还是分层架构图。

---

## 2. 先选视觉语法（不要一张图里混两种世界观）

建议在 Fig00（全览分层图）阶段做二选一：

### A) **分层架构图（推荐默认）**
- 适用：系统分层、职责边界、模块清单、读者是评审/管理/跨团队协作。
- 表达手段：
  - 用 **Layer Card（分层卡片）** 表示层级；
  - 用 **每层 1 根递进箭头** 表示“从上到下的处理链路/抽象递进”；
  - **不使用 connector 连线**（避免 hairball）。
- 验收方式：
  - 视觉：一眼能读出 4 层与每层 4~6 个节点；
  - 程序：`check_pptx_connectors.py` 应该显示 `connectors=0`（或显式设置 `--min-connectors 0`）。

### B) **数据流/因果图（仅在确有需求时）**
- 适用：你必须表达“谁产出什么、谁消费什么”、反馈闭环、控制流、关键依赖。
- 表达手段：
  - connector 只画主线与必要支线；
  - 边必须能被图例或一句话解释；
  - 强制校验粘连（`begin_connect/end_connect` + 校验脚本）。

混用禁忌：
- 分层架构图上画大量数据流边，等价于把两张图硬塞一张。
- 正确做法：**Fig00 分层** + **Fig01 主流程** + **Fig02 角色流程**。

---

## 3. 连线策略：edge budget + 语义分层

### 3.1 Edge budget（建议值）
- Fig00（分层）：0~3 条（只允许“层级递进箭头”或 1 条主闭环）。
- Fig01（主流程）：8~12 条（每条边都必须解释）。
- Fig02（角色协同）：6~10 条（角色到界面/审批点，不要连到所有模块）。

### 3.2 每条边必须回答一个问题
建议用下面的句式自检：
- “A 产出 X，B 消费 X”（数据流）
- “A 触发 B”（控制流）
- “A 审批/约束 B”（治理流）
- “A 反馈到 B”（闭环）

如果一句话说不清这条边的语义，就先删。

---

## 4. 配色：用对比度做底线，用调色盘做一致性

### 4.1 文字颜色：不要用阈值，用对比度
做法：对每个填充色 `fill_rgb`，在候选文字色 `{#192638, #FFFFFF}` 中选择对比度更高的一个（WCAG 思路）。

经验：
- 小字号（≤ 12pt）建议对比度更苛刻；如果必须上深底色，请让标题条面积小、文字变大。

### 4.2 推荐调色盘（4 层 + 中性色）
（示例，偏 Tailwind 风格，适合浅底卡片）
- L1 Data：`#2563EB`
- L2 Research：`#10B981`
- L3 Portfolio：`#F59E0B`
- L4 Execution：`#7C3AED`
- Background：`#F8FAFC`
- Neutral line：`#64748B`
- Neutral fill：`#94A3B8`

### 4.3 面积原则
- **大面积使用低饱和**（tint 到接近白），小面积（标题条/强调）用高饱和。
- 节点建议统一白底 + 彩色描边（而不是每层不同底色），否则会“花”。

---

## 5. 箭头与层级：让箭头是“结构”，不是“装饰”

推荐结构（分层架构图）：
- 每层一个 Card（带标题条）。
- Card 之间只放一个居中的 `DOWN_ARROW`。
- 箭头不写长句：最多 6~10 个词；更长放到旁注或图注。

避免：
- chevron 横向铺满 + 长文案（会显得廉价且信息负担重）。

---

## 6. 验收清单（务必执行）

### 6.1 视觉验收（人工）
- 3 秒测试：能否读出层数、每层 3~6 个节点、主递进方向。
- 30 秒测试：能否说清“这张图回答的问题”。

### 6.2 程序验收（自动）
- 如果图使用 connector：`check_pptx_connectors.py --min-connectors 1`（或更高）并要求 0 error。
- 如果图不用 connector：`--min-connectors 0`，确认 `connectors=0` 与 0 error。

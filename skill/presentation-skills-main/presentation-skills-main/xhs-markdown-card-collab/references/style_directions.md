# XHS Style Directions

## 用途

这份文档回答“如何避免小红书卡片风格趋同”，同时明确：**不要用大改字体大小来制造变化**。
风格的变化应当来自主题、布局重心、信息组织、装饰语言和叙事语气。

## 先固定，再变化

默认先固定这些不轻易动的东西：
- `1080x1440` 竖图
- 已验证的字号合同
- 浏览器真实排版
- 显式 front matter 封面

然后再沿着下面这些轴做变化。

## 风格变化轴

### 叙事姿态

根据内容任务，先决定封面应该像哪一类页面：

- **机构招募型**：强调可信度、层级和正式感，适合招聘、招生、实验室招募、研究合作。
- **研究摘要型**：强调结论、方法、关键发现，适合论文摘要、读书笔记、研究框架。
- **产品说明型**：强调问题、方案、模块、收益，适合产品介绍、工具解释、功能拆解。
- **观点备忘型**：强调一条主结论与几条 supporting points，适合评论、分析、观察。
- **数据卡片型**：强调数字、标签和结论密度，适合榜单、周报、快评。

### 色彩路线

避免所有内容都长成同一套“米白底 + AI 味紫色高光”。

优先考虑这些方向：
- **暖纸 + 青绿 / 暖橙**：适合研究、机构、招聘、专业笔记。
- **矿物灰 + 静蓝 / 砖红**：适合政策、产业、深度分析。
- **深色 editorial**：适合观点、夜读、长标题强封面，但要额外检查对比度。
- **清亮技术色**：适合产品 walkthrough、工具介绍、数据摘要。

除非用户明确要求，不要默认走紫色主导的“通用 AI 风格”。

### 封面信息组织

风格差异可以通过封面结构做出来，而不是只靠换颜色。

可变化的杠杆包括：
- headline 是否承担全部抓眼任务
- `role_line` 是偏口号还是偏交付信息
- badges 是 2-3 个大标签还是 4-5 个紧凑标签
- highlights 是 2 格、4 格还是单列重点块
- organization 是弱存在还是强存在

但无论怎么变，首页都应同时回答：
- 这是什么
- 为什么值得看
- 关键信息在哪

### 装饰语言

优先通过这些元素做差异化：
- 背景光斑位置和强弱
- grain 密度
- 分割线的长度和存在感
- section marker 的形状
- badge 的圆角、边框和底色节奏
- 编号 chip 的视觉语气

不要靠加入大量无意义装饰图形来假装有风格。

## 推荐风格方向

### 机构招募 / 招生

- 标题可以非常强
- organization 保持清晰可读
- highlights 直接给时间、名额、联系方式
- badge 数量适中，不要把首页做成 tag 墙
- 颜色优先稳重、可信、带少量活力

### 研究框架 / 论文摘要

- 标题不一定要最长，但需要主问题很清楚
- highlights 更适合放方法、数据、结论
- 正文层级要更严谨，避免过多营销式强调
- 可让编号列表更强，帮助读者顺着论证走

### 产品 / 工具解释

- 封面可以更强调问题和方案
- badge 可以承载模块名或能力名
- highlights 更适合放输入、输出、速度、适用场景
- 正文适合更多列表和步骤型结构

### 数据快评 / 榜单

- 标题可以更短、更有 punch
- highlights 应当直接承接核心数字
- badge 可以弱化，避免抢数字的注意力
- 面板可更紧凑，但不要牺牲正文可读性

## 防趋同检查

每次出图前问自己：

1. 这次的风格变化是不是只靠改字体大小实现的。
2. 这次的主题色是不是又回到默认的 AI 紫色。
3. 这次的封面是不是仍然是同一套“标题 + badges + 四宫格 highlights”，但内容任务其实不同。
4. 这次的正文是不是只是安全、平均、无重点的排版，没有体现内容类型。

只要前三个问题里有两个回答为“是”，就说明风格已经开始趋同。

## 新主题扩展建议

如果要扩展新主题，优先新增 **token 化主题 preset**，不要直接在单个项目里写临时 CSS。
新主题至少显式定义：
- `bg1/bg2/bg3`
- `ink/muted`
- `accent/accent2`
- `panel/panel2`
- `rule/shadow`

保持 typography 合同不变，优先通过 token 和装饰结构做差异化。

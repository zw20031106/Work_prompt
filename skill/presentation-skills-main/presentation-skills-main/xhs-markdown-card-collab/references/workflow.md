# XHS Markdown Card Workflow

## 用途

这份文档定义小红书 Markdown 卡片的默认工作流、front matter 结构、宿主 CLI 契约和 QA 顺序。
优先把它当作“如何稳定交付卡片 bundle”的操作说明，而不是当作视觉灵感板。

## 默认输入输出

**输入**
- 一份 Markdown 正文，或一份需要轻量语义整理的文本稿。
- 可选 front matter，用于显式控制封面信息。

**输出**
- 逐页 `PNG`
- 一份 `_preview.html`
- 一份 `_metadata.json`

如果宿主实现暂时做不到 HTML 或元数据输出，应该显式说明，不要静默跳过。

## 清晰度与导出

**版式合同** 继续按 `1080x1440` 的浏览器 CSS 像素维护，这能保证已经验证过的字号、分页和留白合同不漂移。

**清晰度合同** 应通过高分辨率导出单独解决。稳定做法是：
- 页面布局仍按 `1080x1440` 计算。
- 导出时使用浏览器截图 supersampling，默认建议 `2x`，对应实际 PNG `2160x2880`。
- PNG 应显式写入 `350 DPI` 元数据，方便后续进入设计、排版或打印链路。

**决策原则** 是先分离“布局问题”和“清晰度问题”。看起来发虚时，优先检查导出像素与截图倍率，不要先动字体、边距或主题。

## 默认 front matter

优先使用下面这类结构。`xhs.cover` 只负责首页，不参与正文分页。

```yaml
---
title: 港科大（广州）人工智能 / 金融科技学域团队招收全奖 PhD Student、RA、Intern
xhs:
  footer: HKUST(GZ) AI / FinTech
```

## 正文整理规则

优先做“轻量语义整理”，不要默认做“重写文案”。

### 应该做

- 给文档补 `#` 和 `##` 标题层级。
- 把 `·` 或非标准 bullet 改成标准 `-` 列表。
- 给极少量关键数字、金额、时间或邮箱加 `**加粗**`。
- 把孤立的小节标题提升为标准 Markdown 标题。

### 不应该默认做

- 大幅重写句子，尤其不要替换原文业务表达。
- 为了缩短页面而拆散作者原本的核心表述。
- 把研究或招聘文本过度改成营销口吻。
- 频繁使用粗体，导致正文全部变成重点。

## 推荐执行命令

如果宿主实现是 Python CLI，优先保持下面这种可复用形态：

```bash
python3 -m xhs_md_cards render INPUT.md \
  -o temp/xhs_cards/output \
  --theme lumen
```

常见迭代手段：
- 换视觉主题：`--theme lumen|clay|ink`
- 限制出图数量做快排：`--max-cards 3`
- 临时关闭封面测试正文：`--no-cover`
- 提高导出锐度：`--export-scale 2 --output-dpi 350`

## QA 清单

渲染后至少逐页检查这些问题：

1. 封面是否承担了任务定义，而不是只剩一个空标题。
2. 封面标题是否完整显示，是否出现压坏 badge 或 highlight 的情况。
3. 页眉、页脚和背景外框是否占用过多面积。
4. 是否出现章节标题孤立在页底，正文在下一页才开始。
5. 列表编号是否连续、缩进是否稳定。
6. 长英文串、邮箱和数字是否撑坏行宽。
7. 是否存在明显过空页或被过度压缩的页。
8. 是否有“字体没溢出，但信息读起来很累”的视觉假成功。
9. PNG 实际像素是否达到目标，例如 `2160x2880`，而不是只有 `1080x1440`。
10. 如果声称是高精度导出，元数据里是否真的写入了目标 DPI。

## 常见返工顺序

遇到问题时按这个顺序修：

1. 修 front matter 和正文语义。
2. 修分页策略，尤其是标题与后续段落绑定。
3. 修封面信息密度。
4. 修外边距和面板 padding。
5. 最后才微调字号，而且只在小范围内调。

## 历史经验教训

### 封面信息太稀

单纯把封面做成“大标题 + 一句工具说明”会导致首页信息密度过低，尤其是招聘、研究摘要、产品说明这类任务。正确做法是通过 front matter 补 `role_line`、`badges` 和 `highlights`，而不是先压缩标题或加一堆正文摘抄。

### 正文改得太多

曾经为了“更像发布文案”而重写正文，结果虽然排版更顺，但语义已经偏离原文。后来的稳定做法是只补章节标题、标准列表和少量加粗。

### 标题孤立在页底

如果分页器只按高度切分 block，很容易出现小节标题落在页底、正文跑到下一页。应优先使用“标题与下一段绑定”的分页规则。

### 外边距太宽

如果背景和白色卡片之间留白过大，第一页会显得发虚、信息密度低。应优先缩窄外边距和内容面板边距，而不是减小正文字号。

### 把 DPI 当成清晰度本身

曾经最容易犯的错误是看到导出图不够锐，就直接要求“350 DPI”。真正决定手机端清晰度的是实际像素和截图倍率。只写 DPI 标签而不提高导出像素，只会得到一个元数据更漂亮但视觉上没有变清楚的 PNG。

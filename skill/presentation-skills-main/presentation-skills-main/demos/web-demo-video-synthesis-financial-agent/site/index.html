Research Demo · Frame-Based Investment Agent

FramePilot：帧驱动的金融 Agent

不追求“猜中一个点”，而是把投资决策拆成可审计、可复盘、可做对照实验的 frames。

可解释
每条结论都能回溯到证据、口径与假设，天然支持审稿式追问。
可复盘
同一份数据、同一份 frame 图，能复现同一条建议与失败路径。
可扩展
frame 是模块化单元：研究者可以替换子模块并做 ablation。
向下滚动 · 跟随旁白节奏

问题：投研不是单目标优化

输入是噪声
  • 宏观变量口径不一致、滞后与修订
  • 新闻是强噪声通道，且易被叙事带偏
  • 资产价格包含预期，信号常常互相抵消
目标是多维
  • 收益、回撤、换手、成本、约束同时存在
  • 风险不是一个数字：尾部、相关、流动性都要管
  • 更关键：研究过程必须能被审计与复现
Agent 的核心价值
把“想法 → 证据 → 决策 → 风控 → 复盘”变成一条可追溯的链，而不是一次性输出。

架构：三层 + 可审计日志

数据层
市场数据 · 财报 · 宏观 · 事件

审计日志（示意）
frame_id: REGIME-2026Q1
evidence:
  - rates: "10Y ↑, term premium ↑"
  - credit: "IG spreads stable"
constraints:
  - max_drawdown: 8%
  - turnover: < 20% / month
decision:
  - tilt: "quality + short duration"
  - hedge: "rate beta neutral"
fail_fast:
  - missing: "earnings calendar"

Frames：把决策拆成“可对照实验”的单元

一个 frame 是什么？
目标、约束、市场 regime、证据与反证，合成一张决策卡片。它既是 reasoning 的单位，也是实验的单位。
为什么要图结构？
真实决策里，证据互相依赖。frame 图能表达依赖、冲突与权重，方便做可重复的“删一条证据再跑”的 ablation。
输出不是一句话
输出包含：建议、置信区间、关键假设、以及“如果假设被推翻该怎么做”的应对分支。

Guardrails：正确地失败，而不是“看起来成功”

失败即信号
  • 数据缺失：明确指出缺的字段与时间范围
  • 口径冲突：写出冲突来源，并停止合成结论
  • 风险超标：输出超标原因与最低改动的修复方案

评估：不仅是收益，更是“过程指标”

收益/回撤
Return · MaxDD
传统指标，必须有，但不够。
证据覆盖率
Evidence Coverage
关键变量是否被显式检查与引用。
反事实一致性
Counterfactual
删掉/替换证据后，结论是否合理变化。
失败可诊断
Diagnosability
失败是否给出可操作的修复路径。

# Lightweight Mode

**这份文档的定位。** 本文是 `word-polished-doc-collab` 的轻量模式入口，只保留你最常见、最刚性的格式要求，以及一个足够简单的 workspace 约定。它服务“先把文档转顺、转美观、快速可交付”的任务，不默认引入质量 gate、默认 review 和重型基础设施。

## 什么时候走轻量模式

**轻量模式适合快任务。** 典型场景包括：
- 单篇或少量文档快速转 `docx`
- 规则已经明确，重点是把版式落对
- 当前不需要 Office 原生 chart / illustration、复杂模板继承、preset 级还原、OOXML patch、自动 QA

**preset 请求默认不归轻量模式。** 如果用户明确要求英文咨询 preset、投资报告 preset、模板继承或咨询报告级图文系统，应默认升级到精细模式；只有用户明确接受“轻量近似、不追求完整 preset 契约”时，才继续留在轻量模式。

## 最小 workspace

**轻量模式的 workspace 可以很简单。** 推荐结构：

```text
doc-job/
├── doc.md
├── assets/
└── out/
```

**如果任务是从原始 Word 起步，可以再加一个输入文件。** 推荐结构：

```text
doc-job/
├── input.docx
├── doc.md
├── assets/
└── out/
```

**轻量模式不强制 `meta.json`、不强制 `temp/`、不强制验证目录。** 先把 Markdown 和最终 `.docx` 跑顺，再决定要不要升级到精细模式。

**轻量模式默认也不强制 review 记录。** 只有用户明确要求自动 review、人工复核或任务已经升级成精细模式时，才追加检查产物。

## 你的默认格式要求

**默认字体 profile 是 `中文宋体 + 英文 Times New Roman`。** 正文、普通标题、表题、图题、表注默认都从这套 profile 出发。

**正文默认使用小四。** 具体要求是：
- 中文字体：`宋体`
- 英文字体：`Times New Roman`
- 字号：`小四 12pt`
- 首行缩进：`2` 字符
- 行距：`1.5`
- 段前：`0.5` 行
- 段后：`0.5` 行

**标题和题注不继承首行缩进。** 标题、表题、图题、表注、列表和表格正文都不应机械沿用正文的 `2` 字符缩进。

**标题按重要性逐级放大。** 轻量模式推荐梯度：
- 文档主标题：`22pt`
- 一级标题：`16pt`
- 二级标题：`15pt`
- 三级标题：`14pt`
- 四级标题：`12pt`

**标题和正文共享段落节奏。** 标题默认也是：
- 行距：`1.5`
- 段前：`0.5` 行
- 段后：`0.5` 行

**表格使用独立密度。** 具体要求是：
- 默认表格字号：`五号 10.5pt`
- 密表回退字号：`小五 9pt`
- 表头：`居中对齐`
- 左侧索引列：`左对齐`
- 右侧数值列：`右对齐`
- 段前：`0`
- 段后：`0`

**表题在表上方。** 具体要求是：
- 位置：表格前一段
- 写法：直接写成 `表 1 费用分摊明细`
- 字号：默认 `12pt`
- 字重：`Bold`
- 段前：`0.5` 行
- 段后：`0`

**图题在图下方。** 具体要求是：
- 位置：图片后一段
- 写法：直接写成 `图 1 费用审批流程`
- 字号：默认 `10.5pt`
- 段前：`0`
- 段后：`0.5` 行

**表注在表下方。** 具体要求是：
- 位置：表格后一段
- 字号：默认 `10.5pt`
- 段前：`0`
- 段后：`0.5` 行

## 可选字体组合

**正式强调可以切到 `中文楷体 + 英文 Times New Roman`。** 这套组合适合引用、签批说明、法规摘录和局部强调，不建议把整篇连续正文都切成楷体。

**展示型标题可以切到 `中文黑体 + 英文 Arial`。** 这套组合适合封面、章节标题、图表短标题和标签，不建议把整篇正式正文都切成黑体。

## 轻量模式的执行原则

**先把语义写清楚，再做版式。** Markdown 里至少要分清标题、正文、表格、图片、表题、图题和表注。表题和图题优先直接写最终交付文本，不要在源稿里保留 `表题：` 这种不属于正式文面的噪声前缀。

**先走最简单的构建路线。** 默认优先 `markdown -> docx` 或 `docx -> markdown -> docx`，不要一开始就引入复杂 OOXML patch。

**轻量模式默认不做 review。** 不默认引入质量 gate、字体槽位检查、人工视觉复核或单独验证目录。只有用户主动要求 review，或者任务已经明显升级成精细模式，才追加这些动作。

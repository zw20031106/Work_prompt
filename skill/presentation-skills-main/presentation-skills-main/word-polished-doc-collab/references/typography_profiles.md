# Typography Profiles

**这份文档的定位。** 本文定义 `word-polished-doc-collab` 的默认字体组合、字号梯度、首行缩进、行距、段前段后、caption policy 与表格密度规则。它是这个 skill 最核心的版式契约。

## 默认 profile

**默认 profile 是 `cn_song_times`。** 除非任务另有明确要求，整份文档默认使用：
- 中文字体：`宋体`
- 英文字体：`Times New Roman`
- 阿拉伯数字：`Times New Roman`

**正文是唯一主档位。** 正文默认使用：
- 字号：`小四 12pt`
- 首行缩进：`2` 字符
- 行距：`1.5`
- 段前：`0.5` 行
- 段后：`0.5` 行

**实现层统一按 `6pt` 落地 `0.5` 行。** Word、WPS、OOXML 和不同脚本库对“按行设置段前段后”的稳定性较差，因此推荐统一把 `0.5` 行转换为 `6pt`。

**实现层统一按 `2em` 落地首行缩进。** 中文正式文档正文默认应有首行缩进，标题、题注、列表和表格正文不继承这个缩进。

## 标题梯度

**标题随重要性递增，不靠猜。** 推荐默认梯度如下：

| role | 推荐字号 | 字重 | 默认字体 |
| --- | --- | --- | --- |
| 文档主标题 `doc_title` | `22pt` | Bold | `宋体 + Times New Roman` |
| 副标题 `subtitle` | `15pt` | Regular | `宋体 + Times New Roman` |
| 一级标题 `heading_1` | `16pt` | Bold | `宋体 + Times New Roman` |
| 二级标题 `heading_2` | `15pt` | Bold | `宋体 + Times New Roman` |
| 三级标题 `heading_3` | `14pt` | Bold | `宋体 + Times New Roman` |
| 四级标题 `heading_4` | `12pt` | Bold | `宋体 + Times New Roman` |

**标题默认和正文共享段落节奏。** 除文档主标题可根据封面风格单独调节外，标题默认使用：
- 行距：`1.5`
- 段前：`0.5` 行
- 段后：`0.5` 行

## 表格体系

**表格有独立密度，不继承正文。** 推荐规则如下：
- 默认表格正文：`五号 10.5pt`
- 密表回退档：`小五 9pt`
- 行距：`单倍`
- 段前：`0`
- 段后：`0`

**表头可以加粗，但不要放大。** 表头的职责是区分列语义，不是再造一套标题层级。默认保持同字号加粗即可。

**表格对齐要体现字段语义。** 推荐规则如下：
- 表头默认 `居中对齐`
- 左侧索引列、条目列、描述列默认 `左对齐`
- 数值列、比例列、金额列默认 `右对齐`
- 非数值且不属于索引列的中间文本列默认 `左对齐`

**密表的使用条件必须明确。** 只有当列数较多、页宽紧张、正文五号已经明显溢出时，才允许降到 `小五 9pt`。

## Caption Policy

**`caption_policy` 是 style profile 的组成部分。** 默认中文正式文档和咨询报告 preset 可以拥有不同的题注位置，但都必须显式写出来。

**`cn_song_times` 的表题在表上方。** 推荐规则：
- role：`table_title`
- 位置：表格前一段
- 文面：直接写 `表 3 情景分析摘要`
- 字号：默认 `12pt`
- 字重：`Bold`
- 行距：`1.5`
- 段前：`0.5` 行
- 段后：`0`

**`cn_song_times` 的图题在图下方。** 推荐规则：
- role：`figure_title`
- 位置：图片后一段
- 文面：直接写 `图 2 成本曲线变化`
- 字号：默认 `10.5pt`
- 行距：`1.5`
- 段前：`0`
- 段后：`0.5` 行

**`cn_song_times` 的表注在表下方。** 推荐规则：
- role：`table_caption`
- 位置：表格后一段
- 字号：默认 `10.5pt`
- 行距：`1.5`
- 段前：`0`
- 段后：`0.5` 行

**`figure_note` 与 `source_note` 默认也在图下方。** 推荐规则：
- role：`figure_note` / `source_note`
- 位置：紧随 `figure_title` 或图像之后
- 字号：默认 `10.5pt`
- 行距：`1.5`
- 段前：`0`
- 段后：`0.5` 行

**preset 可以显式覆盖 `figure_title` 的位置。** 例如红色投资报告 preset 常把 Figure 标题放在图上方，蓝色 editorial article preset 常把 `Exhibit` 标题放在图上方，但这必须写进 preset 配置或其口头定义中。

## 可选字体 profile

**`cn_kaiti_times` 用于正式强调。** 推荐映射：
- 中文：`楷体`
- 英文：`Times New Roman`
- 适用 block：引用、批示、签批说明、法规摘录、局部强调段落

**`cn_heiti_arial` 用于展示型层级。** 推荐映射：
- 中文：`黑体`
- 英文：`Arial`
- 适用 block：封面标题、分节标题、图表短标题、标签、现代商务风格文档

**默认不要把整份正文切成黑体或楷体。** 正文大段连续阅读仍然以 `cn_song_times` 为主，其他 profile 用于明确的风格位点。

## 实现层要求

**字体必须写入全部关键槽位。** 对 Word / OOXML 构建器，至少显式设置：
- `w:ascii`
- `w:hAnsi`
- `w:eastAsia`
- `w:cs`

**`Arial` 作为英文无衬线默认值。** 如果项目需求里把它误写成 `Aril`，实现层应规范化为 `Arial`，不要真的写错字体名。

**字号、缩进和段距要按 role 绑定。** 不要写成“所有非标题都 12pt”这种粗糙规则，否则正文首行缩进、表题、图题、表注与密表无法稳定落地。

**表格对齐要按列角色绑定。** 不要把整张表统一居中或统一左对齐。表头、索引列、文本列和数值列应按语义分别落地。

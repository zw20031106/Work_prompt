# 中文精读卡模板

```markdown
# 论文精读卡

## 输入覆盖范围
- 来源：
- 覆盖级别：全文 / 局部正文 / 仅摘要 / 仅标题与摘要
- 边界说明：

## 标题

## 链接

## 研究问题
- 论文试图解决什么问题？
- 为什么这个问题重要？

## 核心方法
- 整体思路：
- 与已有方法的核心差异：

## 模型 / 算法要点
- 模块或阶段：
- 关键机制：
- 训练或推理上的特殊设计：

## 实验设置
- 任务：
- 数据集：
- 指标：
- baseline：
- 消融或额外分析：

## 主要结果
- 论文声称的主要收益：
- 结果是否充分支持 claim：

## 优势
- 方法层面：
- 实验层面：
- 研究价值层面：

## 局限
- 证据不足处：
- 适用范围限制：
- 可能的失败场景：

## 复现难点
- 数据或环境：
- 训练细节：
- 评测细节：
- 工程复杂度：

## 对当前研究的启发
- 可直接借鉴：
- 可对比验证：
- 可作为反例或边界案例：

## 是否值得精读
- 标签：值得精读 / 值得速读 / 可暂缓
- 判断理由：
```

## 使用说明

- 输入不足时，优先保留结构，不要为了“填满模板”而虚构内容。
- 如果只有摘要，把“实验设置”“主要结果”“复现难点”写得更保守。
- 如果用户是为了决定是否要读全文，把“是否值得精读”的判断写具体。

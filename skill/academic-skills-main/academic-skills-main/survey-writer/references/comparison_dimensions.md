# 常用比较维度

在整理多篇论文时，优先选取稳定、可复用的比较维度。不要机械堆满所有维度，只保留当前主题真正有区分度的项。

## 问题设定维度

- 任务目标是什么？
- 输入输出形式是什么？
- 约束条件是什么？
- 是否面向开放世界、长上下文、多跳推理或安全约束？

## 方法设计维度

- 核心建模思路
- 检索、规划、工具使用或外部记忆是否参与
- 训练方式：监督微调、偏好优化、强化学习、纯推理时方法
- 推理时机制：自一致、树搜索、反思、工具调用、检索增强

## 数据与评测维度

- 使用哪些 benchmark 或数据集
- 指标是否与任务目标真正一致
- 是否有跨数据集泛化、消融、错误分析
- 是否考虑成本、延迟、安全性、鲁棒性

## 研究价值维度

- 是方法创新、数据贡献、评测贡献，还是系统工程整合
- 与已有方法相比到底改进了什么
- 结论是否可迁移到别的场景

## 局限维度

- 对特定模型、数据源或工具生态依赖过强
- 对提示、超参或工程细节敏感
- 证据集中在少量 benchmark
- 缺少失败案例或边界分析

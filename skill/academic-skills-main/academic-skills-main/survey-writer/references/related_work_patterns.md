# Related Work 常用写法

## 模式一：按问题拆分

适合一个主题下存在多个相近但不完全相同的问题设定。

```markdown
现有工作大致可以分为三类：一类关注……，一类重点解决……，另一类则尝试在……约束下实现……。这三类工作的共同目标是……，但在问题假设和评价标准上存在明显差异。
```

## 模式二：按方法路线拆分

适合方法演化清晰的主题。

```markdown
从方法路线上看，相关研究经历了从……到……再到……的演化。早期方法主要依赖……，后续工作开始引入……，最近则更强调……。这种演化反映出研究重点从……转向了……。
```

## 模式三：按证据强度拆分

适合论文结论差异较大、benchmark 支撑不均衡的主题。

```markdown
尽管多项工作报告了性能提升，但其证据强度并不一致。一部分研究在多个 benchmark 上验证了方法有效性，另一部分工作则主要依赖单一数据集或有限设置，因此结论的可迁移性仍需谨慎判断。
```

## 写法提醒

- 避免“已有很多工作研究了……”这类空泛开头。
- 避免把综述写成逐篇摘要拼接。
- 每段 related work 都应回答一个问题：这些工作之间到底怎么分，为什么这样分。
